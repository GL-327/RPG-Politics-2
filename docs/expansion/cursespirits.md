# Cursed Spirits Expansion

Owner package: `com.political.curse.spirits.**` (plus edits to `com.political.curse.ModEntities`,
`CurseManager`, `CurseCommands`). Heavily upgrades the Curses faction with a full Jujutsu-Kaisen
style roster (21 new species across Grade 4 → Special Grade, including named bosses), inspired by
JujutsuCraft's curse line-up and Prominence II mob design. **No mixins** — all behaviour is driven by
vanilla zombie goals + per-tick logic on the server.

## What gets registered, and by whom

### Server (already wired — no integration action needed)
`ModEntities.register()` is already called from mod init. It now also calls
`com.political.curse.spirits.ModSpirits.registerAll()`, which:

- Registers **one `EntityType<CursedSpiritEntity>` per `SpiritSpecies`** into `BuiltInRegistries.ENTITY_TYPE`
  under `politicalserver:<id>` (same pattern as the existing `curse_spirit`).
- Registers each type's **default attributes** via `FabricDefaultAttributeRegistry.register(type, attributesFor(species))`
  (built from `Zombie.createAttributes()` with per-species `MAX_HEALTH`, `ATTACK_DAMAGE`,
  `MOVEMENT_SPEED`, `FOLLOW_RANGE`, `KNOCKBACK_RESISTANCE`, `SCALE`).
- Stores species ↔ type maps so `CursedSpiritEntity` can resolve its species from its own type, and so
  the spawn system can pick a species by grade.

`CurseManager` now drives spawning:
- `spawnAt(level, pos, grade)` → picks a random species suited to the grade (`ModSpirits.randomForGrade`)
  and delegates to `spawnSpeciesAt`.
- `spawnSpeciesAt(level, pos, grade, species)` → creates that species' type (or plain `CURSE_SPIRIT`
  if `species == null`), then `manifest(...)`.
- `manifest(curse, grade)` → named species take a gentler grade ramp (their base stats are pre-tuned),
  apply species colour/name, fire-resistance for fire-immune species, and a visible name + boss bar for
  bosses. Plain `CURSE_SPIRIT` keeps the original steeper scaling.
- `onEntityDeath(...)` → rewards scale by the species **loot tier** and bosses drop extra cursed
  remnants plus a cursed Nether Star.

Natural manifestation (ambient night + cursed-object lures) and the existing `/curse spawn <grade>`
already route through `spawnAt`, so the new roster appears automatically, scaled by grade.

### Client (INTEGRATION MUST BIND THIS)
Add **one line** to `PoliticalClient.onInitializeClient()`, next to the existing
`CurseModels`/`CurseRenderer` registration:

```java
com.political.curse.spirits.SpiritClient.registerClient();
```

`SpiritClient.registerClient()`:
- Registers the **four shared model layers** (`spirit_gaunt`, `spirit_swarm`, `spirit_hulking`,
  `spirit_horned`) via `ModelLayerRegistry.registerModelLayer(...)`.
- Registers a `SpiritRenderer` for **every species' `EntityType`** via `EntityRendererRegistry.register(...)`.

`SpiritRenderer extends HumanoidMobRenderer<CursedSpiritEntity, ZombieRenderState, ZombieModel<ZombieRenderState>>`
(same shape as the existing `CurseRenderer`), bakes the layer for the species' `ModelKind`, and uses
`textures/entity/<id>.png`. The existing `curse_spirit` renderer/layer is untouched.

> The four model layers all ride the humanoid rig; they differ by body bulk (`CubeDeformation`) and
> bolted-on horns / shoulder spikes. Per-grade size is the `SCALE` attribute, so models stay simple.

## Commands (all OP)
- `/curse spawn <grade> [count]` — existing; now manifests random graded species.
- `/curse summon <species> [grade] [count]` — spawn a specific species (tab-completes ids).
- `/curse list` — prints the roster grouped by grade (`*` = boss).

## Textures
Generated by `tools/gen-cursespirits.js` (Node + jimp) into
`src/main/resources/assets/politicalserver/textures/entity/<id>.png` (21 files, 64×64 humanoid skins).
It writes **only** those PNGs. Re-run with `node tools/gen-cursespirits.js` if `tools/node_modules`
exists (it does); already run once.

Translations: `tools/lang-fragments/cursespirits.json` (`entity.politicalserver.<id>` keys) for the
integration agent to merge into `lang/en_us.json`.

## Behaviour notes
- `CursedSpiritEntity extends CurseEntity extends Zombie` → still recognised by
  `CurseManager.isCurse`/`onEntityDeath`, so exorcism rewards/grade progression work unchanged.
- Inherited vanilla goals handle pathing, melee, and targeting (players + villagers). Special traits
  (`Behavior` enum) run in `tick()` server-side with per-trait cooldowns; auras/blasts/shockwaves use
  `level.damageSources().magic()` and `hurtServer(...)`, matching the rest of the codebase.
- Boss bars are self-managed `ServerBossEvent`s updated each tick for players within 42 blocks and torn
  down in `remove(RemovalReason)`.
