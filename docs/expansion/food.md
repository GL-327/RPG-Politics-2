# Food Expansion

Croptopia-inspired roster of **45 edible items** — fruits, vegetables, staple
crops, cooked meals, desserts and drinks. Everything is a plain vanilla `Item`
carrying `FoodProperties` (hunger + saturation); buff foods additionally attach
a `Consumable` component that applies short vanilla status effects on eat/drink.

- **Package:** `com.political.expansion.food`
- **Entry class:** `FoodItems` — exposes `register()` and `items()`
- **Item id prefix:** `food_`
- **Mod id / namespace:** `politicalserver`
- No mixins, no edits to other agents' packages, no crop blocks (food items only).

## Integration (what the integration agent must wire)

1. **Register** during mod init (server/common), alongside the other
   `register()` calls (e.g. next to `RelicItems.register()`):

   ```java
   com.political.expansion.food.FoodItems.register();
   ```

2. **Creative tab** — feed the items into a tab via the existing catalogue
   pattern (mirrors `CreativeCatalog.governance()` which iterates
   `GovItems.items()`):

   ```java
   for (net.minecraft.world.item.Item item : com.political.expansion.food.FoodItems.items()) {
       out.accept(new net.minecraft.world.item.ItemStack(item));
   }
   ```

   Either add this to an existing tab (e.g. "Relics & Consumables") or create a
   new "RPG — Food & Cooking" tab. `FoodItems.items()` returns all 45 items in
   declaration order.

3. **Translations** — merge `tools/lang-fragments/food.json` into
   `assets/politicalserver/lang/en_us.json` (the integration/lang step owns that
   file). Keys are `item.politicalserver.food_*`.

4. **Textures / models** — already generated by `tools/gen-food.js`
   (jimp). Each item has:
   - `textures/item/<id>.png`
   - `models/item/<id>.json` (parent `item/generated`)
   - `items/<id>.json` (item-definition)

   Re-run with `node tools/gen-food.js` if art needs regenerating.

## Notes

- Effects use vanilla `MobEffects` via the 1.21.2+ `Consumable` component
  (`ApplyStatusEffectsConsumeEffect`), so no custom mod systems are required and
  nothing else in the mod is touched.
- All amplifiers are level 0 (i.e. "I") to keep food a gentle, sustained boon
  rather than a combat crutch.
