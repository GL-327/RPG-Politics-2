# Ranged & Magic Weapon Expansion

A self-contained, **mixin-free** arsenal of bows, crossbows, throwing weapons, wands,
staves, spellbooks/tomes and orbs. Every item uses the mod's **Skyblock stat system only**
(no vanilla base attack/armour numbers) and exposes a **RIGHT CLICK spell** implemented via
Fabric's `UseItemCallback` + vanilla mechanics (raycasts, cones, area queries, lightning
entities and particle "projectile" trails).

- **Package:** `com.political.expansion.ranged` (does not touch any other agent's code)
- **Item ids:** all prefixed `arc_` (e.g. `arc_emberbow`)
- **Entry point:** `RangedExpansion.register()` and `RangedExpansion.items()`
- **Stat reuse:** stats live in `custom_data` and resolve through the shared
  `com.political.items.ItemStats` pipeline exactly like `RpgItem` (authoritative explicit
  stats, rarity, `ITEM_MODEL` override, `ItemAttributeModifiers.EMPTY`).

## Integration checklist (for the integration agent)

1. **Register the cast engine** — add one line in `RpgPoliticsMod#onInitialize`, next to the
   other `*.register()` calls (e.g. after `ItemActiveAbilityEngine.register();`):

   ```java
   com.political.expansion.ranged.RangedExpansion.register();
   ```

2. **Show items in a creative tab** — add to `CreativeCatalog#gear` (or a new tab):

   ```java
   for (net.minecraft.world.item.ItemStack s : com.political.expansion.ranged.RangedExpansion.items()) out.accept(s);
   ```

3. **Translations** — merge `tools/lang-fragments/ranged.json` into
   `assets/politicalserver/lang/en_us.json` (item names are also set as literal custom names,
   so this is cosmetic/fallback).

4. **Textures/models** — already generated by `tools/gen-ranged.js` into
   `assets/politicalserver/textures/item/arc_*.png`, `models/item/arc_*.json` and
   `items/arc_*.json`. Re-run with `cd tools && node gen-ranged.js` if needed.

Nothing else is required: cooldowns and Mana are tracked inside `RangedAbilityEngine`, and
the tooltip is built by `RangedTooltip` (so the spell block shows without touching the shared
ability enum).

## How casting works

- Right-clicking with a ranged/magic weapon in the main hand fires its bound `RangedCast`.
- The cast checks an internal cooldown, then spends Mana (`StatManager.spendMana`). If a
  strictly-targeted cast finds no target, the Mana is refunded.
- **Intelligence/Mana scaling:** every cast's power is multiplied by
  `1.0 + max(0, (maxMana - 100) / 300)` — more Intelligence (max Mana) = stronger spells.
- Rarities run Common → Mythic and scale the displayed stat totals via the shared rarity
  multipliers.

## Design notes

- **No mixins, no shared-enum edits.** The cast engine, weapon catalogue, item builder and
  tooltip all live in `com.political.expansion.ranged`. The shared `ItemActiveAbility` /
  `Ability` enums are untouched; this expansion carries its own `arc_cast` marker in
  `custom_data` and resolves spells from its own `RangedCast` enum.
- **Projectiles are hitscan/area + particle trails + lightning entities**, mirroring the
  proven patterns in `ItemActiveAbilityEngine` and `PowerManager` for robustness on the 26.2
  client/server pipeline.
- **Reference influence:** Prominence II (RPG mage classes/spell variety) and the JJK pack
  (energy-projectile feel) — emulated for quality and variety.
