# RPG Creatures & Bosses (expansion mobs)

New, non-curse creature content. All code lives in `com.political.expansion.mobs` (server) and the
matching package under `src/client` (renderers). Nothing here edits another agent's package; it only
references existing public APIs (`DataManager.addCoins`, `RpgItems.create`).

- **No mixins.** One spec-driven entity class (`ExpansionMob extends Monster`) wired purely with
  vanilla AI goals. Behaviour/stats/drops/spawns come from a `MobSpec`.
- **No "dungeon" content.** These spawn naturally in the overworld (and via command), never in
  removed skyblock dungeons.
- 23 creatures: hostile monsters, neutral wild creatures, mini-bosses, and 3 phased bosses.

## What integration must bind

The integration agent must add these three calls (it owns the off-limits files):

1. **Common init** — in `RpgPoliticsMod.onInitialize()`:
   ```java
   com.political.expansion.mobs.ExpansionMobs.register();
   ```
   This registers entity types, default attributes, natural spawns, and the death-reward hook.
   Call it once (order relative to other registrations doesn't matter).

2. **Command callback** — inside the existing `CommandRegistrationCallback.EVENT` lambda:
   ```java
   com.political.expansion.mobs.MobCommands.register(dispatcher);
   ```

3. **Client init** — in `PoliticalClient.onInitializeClient()`:
   ```java
   com.political.expansion.mobs.ExpansionMobsClient.registerClient();
   ```
   This registers the two model layers (`expansion_mob`, `expansion_mob_brute`) and one renderer
   per entity type (each with its own fixed texture).

4. **Translations** — merge `tools/lang-fragments/mobs.json` into
   `assets/politicalserver/lang/en_us.json` (the lang file is off-limits to this agent).

5. **Textures** — generated by `node tools/gen-mobs.js` into
   `assets/politicalserver/textures/entity/mob_*.png` (run if not already present).

No new networking, no new creative-tab entries, no block/item registration are required.

## Commands (`/rpgmob`, op only)

- `/rpgmob list` — list all creature ids.
- `/rpgmob summon <id> [count]` — spawn near you.
- `/rpgmob boss <id>` — spawn a single creature (intended for the bosses).
- `/rpgmob spawns` — toggle natural spawning of newly-spawned creatures.

## Design notes

- Every creature uses `MobCategory.MONSTER` and the standard monster spawn rule
  (`Monster.checkMonsterSpawnRules`, dark + on ground). Neutral "wild" creatures therefore spawn at
  night; this is intentional (dangerous nights). Tighten biome selectors later if desired.
- Drops are awarded by a self-contained `ServerLivingEntityEvents.AFTER_DEATH` listener registered
  in `ExpansionMobs.register()` (coins via `DataManager.addCoins`, items via `ItemEntity`). It does
  not override vanilla loot methods.
- Bosses (`mob_warlord_kael`, `mob_storm_tyrant`, `mob_lich_sovereign`) have a boss bar and a
  phase-2 enrage at 50% HP (speed + resistance, summon adds, plus their signature ability). They do
  **not** spawn naturally — summon them with `/rpgmob boss <id>`.
