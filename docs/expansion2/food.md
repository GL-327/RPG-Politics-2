# Food Expansion — Phase 2 (Global Cuisine)

Croptopia-scale roster of **160 edible items** covering Asian, European,
Middle Eastern, dessert, drink, street-food and hero-tier buff meals. Everything is a
plain vanilla `Item` with `FoodProperties`; 71 items attach a `Consumable`
for short vanilla status effects (20 drinks use the drink animation and are
always edible).

- **Package:** `com.political.expansion2.food`
- **Entry class:** `Food2Items` — exposes `register()` and `items()`
- **Item id prefix:** `food2_`
- **Mod id / namespace:** `politicalserver`
- No mixins, no Phase-1 `com/political/expansion/food/**` edits, no crop blocks.

## Integration (what the integration agent must wire)

1. **Register** during mod init (server/common), next to Phase-1 food:

   ```java
   com.political.expansion2.food.Food2Items.register();
   ```

2. **Creative tab** — iterate `Food2Items.items()` (stable declaration order):

   ```java
   for (net.minecraft.world.item.Item item : com.political.expansion2.food.Food2Items.items()) {
       out.accept(new net.minecraft.world.item.ItemStack(item));
   }
   ```

   Add to an existing consumables tab or create **"RPG — Global Cuisine"**.

3. **Translations** — merge `tools/lang-fragments/food2.json` into
   `assets/politicalserver/lang/en_us.json`. Keys are `item.politicalserver.food2_*`.

4. **Textures / models** — generated by `node tools/gen-food2.js` (jimp). Each item has:
   - `textures/item/<id>.png`
   - `models/item/<id>.json`
   - `items/<id>.json`

   Source roster: `tools/food2-roster.js`. Regenerate Java/lang via
   `node tools/build-food2-java.js`.
